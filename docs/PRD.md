## 1. 專案概述

### 1.1 背景與動機

現代人取得食譜的管道很多（網路、書籍、親友分享），卻常常找不到當初收藏的那道菜。本系統提供一個清楚的食譜收藏與分類平台，讓使用者能方便地新增、整理、查閱自己的食譜清單，解決「食譜散落四處、臨時找不到」的日常痛點。

### 1.2 目標用戶

| 用戶類型 | 描述 |
|----------|------|
| 一般大眾 | 任何想整理並保存日常食譜的人，不限年齡或烹飪程度 |

### 1.3 核心價值主張

> 「清楚的收藏及分類食譜，讓做菜變得更輕鬆。」

- 🗂️ **一鍵收藏**：快速儲存喜愛的食譜
- 🏷️ **清楚分類**：依料理類型輕鬆歸類與篩選
- 📋 **詳細呈現**：逐步驟說明 + 完整食材清單，人人都能上手

## 2. 功能需求

### 2.1 主要功能列表

| # | 功能名稱 | 說明 |
|---|----------|------|
| F1 | 食譜收藏（新增） | 使用者可輸入名稱、描述、分類等基本資訊，新增一筆食譜 |
| F2 | 食譜分類管理 | 提供類別標籤（早餐、午餐、晚餐、甜點、飲品等），可依類別瀏覽 |
| F3 | 食材清單 | 每道食譜列出所需食材名稱、份量與單位，清楚易讀 |
| F4 | 烹調步驟 | 每道食譜以編號步驟逐一說明烹調方法，方便邊看邊做 |
| F5 | 食譜詳細頁面 | 整合食材與步驟，顯示完整食譜資訊（含烹調時間、份量、難易度） |
| F6 | 食譜搜尋與篩選 | 依關鍵字或分類快速找到目標食譜 |
| F7 | 食譜編輯與刪除 | 允許修改既有食譜或刪除不需要的收藏 |

### 2.2 使用者故事

**F1 — 食譜收藏（新增）**
> 作為一般使用者，我希望能填寫食譜名稱與基本資訊來新增收藏，以便日後隨時查閱。

**F2 — 食譜分類管理**
> 作為一般使用者，我希望能為食譜指定類別（如：晚餐、甜點），以便依情境快速找到合適的料理。

**F3 — 食材清單**
> 作為一般使用者，我希望看到每道食譜完整的食材清單與份量，以便準備材料時不遺漏任何東西。

**F4 — 烹調步驟**
> 作為一般使用者，我希望食譜能以清楚編號的步驟呈現，以便下廚時一步一步跟著做。

**F5 — 食譜詳細頁面**
> 作為一般使用者，我希望點選食譜後能看到所有完整資訊（食材 + 步驟 + 時間 + 份量），以便一頁內掌握所有需要的資訊。

**F6 — 食譜搜尋與篩選**
> 作為一般使用者，我希望能用關鍵字或點選類別來過濾食譜清單，以便快速找到目標食譜。

**F7 — 食譜編輯與刪除**
> 作為一般使用者，我希望能更新食譜內容或刪除不再需要的食譜，以便保持收藏庫的整潔。

## 3. 非功能需求

### 3.1 技術限制

| 項目 | 規格 |
|------|------|
| 後端框架 | Python Flask |
| 模板引擎 | Jinja2 |
| 資料庫 | SQLite |
| 前端 | HTML5 + Vanilla CSS + 少量 JavaScript |
| 部署環境 | 本機開發（localhost） |

### 3.2 效能需求

- 頁面載入時間應在 **2 秒內**完成（本機環境）
- 搜尋結果回應時間在 **1 秒內**

### 3.3 安全與資料考量

- 使用者輸入透過 Jinja2 自動 escape，防止 XSS
- 資料庫操作使用參數化查詢，防止 SQL Injection
